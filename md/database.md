# Expense Tracker Database Documentation

## 🎯 Purpose

The Expense Tracker application uses MongoDB as its database.

MongoDB stores data in:

* Databases
* Collections
* Documents

Unlike MySQL, MongoDB does not use tables.

Instead it stores JSON-like documents.

---

# 📋 Database Overview

## Database Name

```text
expense_tracker
```

Created automatically when first data is inserted.

---

# Database Structure

```text
expense_tracker
│
├── users
│
├── expenses
│
└── categories
```

---

# 🧑 USERS COLLECTION

## Purpose

Stores user account information.

Every registered user has one document.

---

## Collection Name

```text
users
```

---

## Sample Document

```json
{
    "_id": ObjectId("687123abc"),

    "name": "Shubham",

    "email": "shubham@gmail.com",

    "phone": "9876543210",

    "username": "shubham",

    "password": "$pbkdf2:sha256:...",

    "budget": 50000
}
```

---

## Field Explanation

### _id

Type:

```text
ObjectId
```

Purpose:

Unique identifier generated by MongoDB.

Example:

```text
687123abc
```

---

### name

Type:

```text
String
```

Purpose:

Stores user's full name.

---

### email

Type:

```text
String
```

Purpose:

Stores email address.

---

### phone

Type:

```text
String
```

Purpose:

Stores mobile number.

---

### username

Type:

```text
String
```

Purpose:

Used during login.

Must be unique.

---

### password

Type:

```text
String
```

Purpose:

Stores hashed password.

Never stores plain text password.

Example:

```text
pbkdf2:sha256...
```

---

### budget

Type:

```text
Number
```

Purpose:

Monthly spending limit.

Example:

```text
50000
```

---

# 💰 EXPENSES COLLECTION

## Purpose

Stores all user expenses.

Each expense becomes one document.

---

## Collection Name

```text
expenses
```

---

## Sample Document

```json
{
    "_id": ObjectId("789xyz"),

    "user_id": "687123abc",

    "expense_name": "Pizza",

    "amount": 350,

    "category": "Food",

    "date": "2026-06-20",

    "notes": "Dinner with friends",

    "created_at": "2026-06-20",

    "updated_at": "2026-06-20"
}
```

---

## Field Explanation

### user_id

Type:

```text
String
```

Purpose:

Links expense to owner.

Example:

```text
user_id = session["user_id"]
```

This ensures:

User A cannot see User B's expenses.

---

### expense_name

Type:

```text
String
```

Example:

```text
Pizza
```

Purpose:

Name of expense.

---

### amount

Type:

```text
Number
```

Example:

```text
350
```

Purpose:

Expense value.

---

### category

Type:

```text
String
```

Example:

```text
Food
```

Purpose:

Expense grouping.

---

### date

Type:

```text
String
```

Example:

```text
2026-06-20
```

Purpose:

Expense date.

---

### notes

Type:

```text
String
```

Purpose:

Optional remarks.

Example:

```text
Dinner with friends
```

---

### created_at

Purpose:

Tracks creation time.

---

### updated_at

Purpose:

Tracks modification time.

---

# 📁 CATEGORIES COLLECTION

## Purpose

Stores custom categories.

Categories are user-specific.

---

## Collection Name

```text
categories
```

---

## Sample Document

```json
{
    "_id": ObjectId("cat123"),

    "user_id": "687123abc",

    "category_name": "Food",

    "created_at": "2026-06-20"
}
```

---

## Field Explanation

### user_id

Purpose:

Links category to owner.

Without this field:

All users would share categories.

---

### category_name

Examples:

```text
Food
Travel
Shopping
Bills
Education
Entertainment
```

Purpose:

Expense classification.

# 🔒 Security Design

## User Isolation

Every query uses:

```python
"user_id": session["user_id"]
```

Example:

```python
expenses_collection.find({
    "user_id": session["user_id"]
})
```

Benefits:

* Data Privacy
* Multi-user Support
* Security

# 📊 Report Calculations

## Total Spending

Formula:

```text
Sum(All Expense Amounts)
```

---

## Remaining Budget

Formula:

```text
Budget - Total Spending
```

Example:

```text
50000 - 12000
=
38000
```

---

## Budget Usage %

Formula:

```text
(Total Spending / Budget) × 100
```

Example:

```text
12000 / 50000 × 100

=
24%
```
